Source-level module walkthrough

Implementation details for every active code/configuration area: entry points, data structures, endpoints, validation boundaries, persistence, concurrency, deployment behavior, tests, and known accuracy limitations.

Five upstream identities normalize to one canonical JWT.

This page documents what the source does now. It does not silently promote target architecture statements into implemented behavior. File paths cite the project's private source repository — they are provenance, not links. Generated data, local secrets, certificates, caches, lockfiles, and compiled output are runtime artifacts rather than functional modules.

Implementation baseline

MCP runtime

Both servers use @modelcontextprotocol/sdk@1.29.0, Express 5, Zod 3, and JOSE 6. They create a fresh stateless StreamableHTTPServerTransport for every POST. No session ID is generated.

This is the 2025-era SDK in stateless mode. It does not implement the 2026-07-28 per-request metadata/header era.

OAuth runtime

Keycloak 26.3 is the hub. Resource servers validate JWTs using remote JWKS, pin algorithms to PS256/ES256, check issuer and resource audience, and publish RFC 9728-shaped metadata.

The lab uses HTTP and mcp:// resource identifiers; those are not production RFC 9728 identifiers.

Data services

HAPI FHIR 8.0.0 serves R4 data; PostgreSQL 16 stores Keycloak, HAPI, and scheduling state; OpenBao 2 stores vendor credentials; Kong Gateway 3.9 runs hybrid data planes.

Observability

OpenTelemetry Collector 0.115.1 forwards traces to Tempo 2.6.1 and logs to Loki 3.3.2. Alloy 1.5.1 tails container logs. Grafana 11.4.0 provisions data sources and a tuple dashboard.

FHIR clinical MCP server

Patient-compartment FHIR reads and stateless scheduling holds.

src/index.tsentry point

Re-exports the server and OAuth module, imports startServer, and starts the Express listener immediately. Docker executes compiled dist/index.js.

src/mcp-server.tscore server

Defines environment configuration, five embedded tool descriptors, JSON-Schema-to-Zod conversion, upstream FHIR calls, tool registration, scope middleware, Origin/Host guard, PRM routes, and HTTP transport lifecycle.

src/oauth-resource-server.tsJWT + PRM

Builds resource metadata and creates an SDK OAuthTokenVerifier. JOSE checks JWKS signature, issuer, expiration, configured audience, and PS256/ES256 algorithm. Additional checks require contract 1.0, exactly one tier audience, and prohibit a patient claim outside Auth0-origin tokens.

It does not enforce every contract-mandatory claim (e.g. groups/amr semantics stay at the DP), but it rejects Auth0-origin tokens that lack a non-empty fhir_patient.

src/dpop.tsDPoP re-check

Byte-identical in both servers. dpopSchemeShim rewrites an Authorization: DPoP <t> header to Bearer so the SDK parses it; requireDpop then authoritatively re-validates the proof with JOSE EmbeddedJWK and calculateJwkThumbprint against cnf.jkt. Unlike cnf-check, DPoP is re-checked at the server. Bearer tokens without cnf.jkt are unaffected.

src/authz-hook.tscompartment

If auth.extra.fhir_patient exists, by-ID tools must match it and search tools have their patient argument overwritten. Without that claim, arguments pass unchanged.

src/audit.tsJSON audit

Writes one JSON line containing server, tool, final decision/reason, token ID, principal, acting agent, client, origin, tier, patient compartment, and timestamp.

openapi.jsonsource description

Describes the curated FHIR endpoints. Runtime does not dynamically load it; descriptors are embedded in the TypeScript source.

Persistence and concurrency

Startup
PostgreSQL advisory lock; create/alter table; expire duplicate held rows; recreate partial unique index on held slot IDs.
Hold
Expire elapsed hold, insert with conflict-do-nothing, return existing handle to same subject or deny another subject.
Confirm
Read, verify owner/status/expiry, then update held → confirmed.
Release
Read, verify owner, then update to released.
Model limitations: find-slots never queries holds; confirmed rows do not reserve slots; release can change confirmed rows; confirm does not check whether its UPDATE won a race. This proves externalized handles and active-hold contention, not complete booking semantics.

Scheduling does not validate Origin and defines no explicit GET/DELETE 405 routes.

Kong gateway configuration and plugins

deck/internal.yamldeclarative state

Global OpenTelemetry, certificate-binding, DPoP sender-constraint, and tier-wall plugins. Routes HAPI, FHIR MCP, scheduling MCP, GitHub egress, and mockhub egress. Applies rate limits and route-specific OIDC/ACL/DLP/vendor-token plugins. The tier-wall pre-function accepts both Bearer and DPoP authorization schemes.

deck/external.yamldeclarative state

Independent external tier wall, OTel, a placeholder FHIR ACL route, and patient-facing FHIR MCP route.

cnf-check/handler.luacertificate binding

Requests a client certificate, parses unverified JWT payload only to locate the thumbprint, hashes leaf DER, and returns 401 on missing/unreadable/mismatch. Later components validate signature.

dpop-check/handler.luasender-constraint

When the token carries cnf.jkt, validates the DPoP proof structure, matches the proof key’s RFC 7638 thumbprint to the claim, checks htm/htu/iat/ath, and enforces single-use jti via a dpop_jti shared dict. The replay cache fails closed: a missing dict or a failed insert returns 503, and forcible LRU evictions are logged as a degraded-mode audit event. It does not verify the proof’s ECDSA signature (the DP does no crypto on first-party routes); the MCP server re-checks authoritatively.

dlp-egress/handler.luainspection

Reads raw body, fails closed for unreadable non-GET/HEAD input, executes configured PCRE patterns, blocks on error/match, and logs only the pattern name.

vendor-token/handler.luacredential swap

Posts vendor/sub/JTI/TTL/scopes to the broker, replaces upstream Authorization on 200, maps consent/re-consent responses, and fails closed when broker resolution fails.

plugins/*/schema.luaschemas

Declare valid configuration fields. Setup scripts register these schemas with Konnect before decK sync.

Resolve/refresh algorithm
  1. Validate hub JWT and use its subject.
  2. Validate vendor and requested scopes against ceiling.
  3. Read cache/Vault; absent or stale means consent.
  4. Missing scopes means re-consent for a capped union.
  5. Return token when remaining TTL is sufficient.
  6. Acquire per-vendor/subject asyncio lock and reread.
  7. Refresh once and persist with Vault CAS plus generation increment.
  8. Invalid grant marks STALE; vendor/Vault failure fails closed.

The lock is single-process. CAS prevents stale writes but does not stop duplicate vendor refresh calls across broker replicas.

Consent state

In-memory transactions carry subject/vendor/scopes/time. State records carry transaction, subject, vendor, PKCE verifier, nonce, issuer, issuer-required flag, consumed flag, and time. Both expire after ten minutes; restart abandons them.

Audit path

Kong OTel → Collector → Loki/Tempo. Service stdout → Alloy Docker discovery → Loki. Grafana provisions both sources and the audit tuple dashboard.

Containers

Node servers use multi-stage builds and run as node. Broker and agent run as UID 65534. Server health checks query protected-resource metadata.

Local artifacts

Environment files, generated DP settings, cert/private keys, Synthea output, caches, lockfiles, compiled output, and dependencies are operational artifacts—not separately documented modules.

CP resiliency mechanism

Nginx carries aliases for Konnect cluster/telemetry endpoints. Stopping it severs DP uplink while leaving cached gateway configuration serving.

Tests, CI, and presentation

Phase tests

Phase 0–7 suites cover claims, normalization, tier isolation, CP loss, decK drift, tools, step-up, compartments, stateless handles, mTLS/cnf, consent, DLP, refresh race, revocation, audit joins, and red-team probes including the DPoP proof suite.

tests/phase3/mcp_http.py

Legacy JSON-RPC helper that sends no modern protocol metadata or mirrored headers. It verifies current application behavior, not 2026 wire compliance.

GitHub Actions

Builds TypeScript, compiles/imports Python, parses JSON/YAML. Production npm audit is non-blocking; live stack gates remain manual.

Coverage required before reassessment

  • server/discover and version negotiation.
  • Required per-request _meta and HTTP header/body binding.
  • Unsupported-version and modern error codes.
  • Malformed but correctly signed contract-token fixtures.
  • Origin-negative coverage for every MCP server.
  • Scheduling availability and confirm/release race tests.
