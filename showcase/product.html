Why this exists

AI arrived faster than the guardrails

Health systems are already full of AI assistants that want to read charts, book appointments, file tickets, and call vendor tools. Three problems show up on day one:

Anonymous hands

An agent acts, something changes in a clinical system, and nobody can say which human set it in motion. In healthcare, an action without an accountable person is a finding, not a feature.

Data that wanders

The fastest way to leak PHI is an eager assistant pasting a chart into a ticket bound for a vendor with no business associate agreement. Good intentions don't survive autocomplete.

Credential sprawl

Every new tool wants a token. Tokens end up in laptops, scripts, and shared accounts — unowned, unrotated, unrevoked when someone leaves. Each one is a standing incident.

The promises

What the platform guarantees

Every action has a name

Human, agent, tool, patient record touched, decision, timestamp — captured on every single call, joinable in one query. An auditor's question becomes a lookup, not a project.

Patient data stays home

Everything leaving for a vendor is screened for patient identifiers before it goes. A chart pasted into a ticket gets blocked and recorded — the vendor never sees it, and you can prove it.

Patients see only themselves

A patient-facing assistant is structurally incapable of reaching another person's record. Not "policy says it shouldn't" — the credential it holds cannot express the request.

Your IdP stays in charge

Which AI clients may talk to which systems, for whom, with what ceiling — decided centrally in the identity provider you already run, the way the MCP Enterprise-Managed Authorization standard defines it. No per-app consent fatigue, no shadow connections.

Nobody carries keys — and a stolen one is inert

No personal access tokens on laptops, no shared service accounts. Vendor credentials are acquired once with the user's consent, kept in a vault, rotated automatically, and revoked the day someone offboards. And the working tokens themselves are bound to the device that earned them, so one lifted from a laptop can't be replayed from anywhere else.

How it feels

Three ordinary moments

The ticket that almost carried a chart

A developer's coding assistant files a GitHub issue about a data bug and helpfully includes a sample — with a real medical record number in it. The platform screens the outbound call, blocks it, and records the verdict. The assistant retries with the identifier redacted; the issue lands.

Blocked and recorded — the vendor never saw the number, and the audit trail shows both the attempt and the clean retry.

The night-shift agent

An autonomous agent reconciles tomorrow's schedule at 3 a.m. It authenticates with a short-lived machine credential — no password, no secret in its config — books what it needs under its own name, and every hold it places is attributable to it, not to a shared account.

Zero secrets, full attribution — the agent works unattended and the morning report shows exactly what it did.

The auditor's question

"Show me everyone and everything that touched this patient's record in March." One query, keyed on the thread that runs through every call: the humans, the agents acting for them, the tools invoked, and each decision along the way — including the denials.

Minutes, not weeks — because the trail was designed in, not reconstructed after.

The governance story

Connections are policy. Tool calls are enforcement.

Enterprise admins decide which AI clients may connect to which systems — centrally, in the identity provider, before a single request flows.

That's the connection layer, and it follows the MCP Enterprise-Managed Authorization standard: sign in once through the enterprise IdP, and sanctioned AI tools connect to sanctioned systems with no per-app consent ceremony. What each request may do is then enforced again at the gateway and inside every tool — least privilege by scope, by role, by patient. Neither layer assumes the other did its job.

And every transitional piece — brokered credentials for vendors that don't speak the standard yet — is built with a defined exit, shrinking as the ecosystem adopts it.
